Deployment Guide

Deploy TwentyCRM
for your group

Three deployment options — from a quick cloud trial to a production self-hosted instance with Cloudflare edge caching. Pick the path that fits your team.

Deployment options

Recommended

Option A: Self-hosted on VPS + Cloudflare

Run TwentyCRM on a VPS (Hetzner, DigitalOcean, GCP) with PostgreSQL and Redis. Front it with Cloudflare for SSL, DDoS protection, caching, and the 2nth.ai domain.

Cost~$20-40/mo (VPS) + free Cloudflare tier
Data residencyYou choose the region (Hetzner Johannesburg available)
Best forProduction use, full control, POPIA compliance
Quick start

Option B: Twenty Cloud (hosted)

Use Twenty's managed cloud at app.twenty.com. No infrastructure to manage. Connect via API and webhooks.

CostFree tier available, paid plans for teams
Data residencyTwenty's infrastructure (US/EU)
Best forQuick evaluation, small teams, no ops overhead
Advanced

Option C: Cloudflare Workers proxy + Twenty API

Run a Cloudflare Worker that proxies and caches the Twenty GraphQL API. The CRM dashboard runs hosted, but your 2nth.ai platform talks to it through a fast edge layer.

CostTwenty Cloud + Cloudflare Workers (free tier)
Data residencyAPI cache at Cloudflare edge, data in Twenty Cloud
Best forLow-latency API access from 2nth.ai platform, no server to manage

Option A: Self-hosted deployment

1

Provision a server

Ubuntu 22.04+, 2 vCPU, 4GB RAM minimum. Hetzner CPX21 in Johannesburg is ideal for SA data residency.

2

Install Docker

curl -fsSL https://get.docker.com | sh
sudo usermod -aG docker $USER
3

Clone and configure

git clone https://github.com/twentyhq/twenty.git
cd twenty/packages/twenty-docker

# Copy and edit environment
cp .env.example .env
nano .env

Key env vars to set:

# .env
SERVER_URL=https://crm.2nth.ai
FRONT_BASE_URL=https://crm.2nth.ai
POSTGRES_ADMIN_PASSWORD=your-strong-password
ACCESS_TOKEN_SECRET=generate-with-openssl-rand-hex-32
LOGIN_TOKEN_SECRET=generate-with-openssl-rand-hex-32
REFRESH_TOKEN_SECRET=generate-with-openssl-rand-hex-32
4

Start TwentyCRM

docker compose up -d

This starts PostgreSQL, Redis, the Twenty server, and the web UI. Takes ~2 minutes on first run.

5

Point Cloudflare

In your Cloudflare DNS for 2nth.ai:

TypeNameTargetProxy
Acrmyour-server-ipProxied

Cloudflare handles SSL termination, DDoS protection, and caching automatically.

6

Connect webhooks

In TwentyCRM settings, add webhook targets:

# Stage change → update 2nth.ai platform
https://2nth.ai/api/crm/webhook

# Payment confirmed → recharge token account
https://2nth.ai/api/billing/webhook

Option C: Cloudflare Workers proxy

If you use Twenty Cloud but want fast API access from the 2nth.ai platform, deploy a Cloudflare Worker that proxies and caches the GraphQL API:

// workers/twenty-proxy.ts
export default {
  async fetch(request: Request, env: Env): Promise<Response> {
    const url = new URL(request.url);

    // Only proxy /graphql
    if (url.pathname !== '/graphql') {
      return new Response('Not found', { status: 404 });
    }

    // Forward to Twenty API
    const twentyUrl = `${env.TWENTY_API_URL}/api`;
    const response = await fetch(twentyUrl, {
      method: request.method,
      headers: {
        'Authorization': `Bearer ${env.TWENTY_API_TOKEN}`,
        'Content-Type': 'application/json',
      },
      body: request.body,
    });

    // Cache read queries for 60s
    const body = await request.clone().json() as any;
    const isQuery = !body.query?.trim().startsWith('mutation');

    const res = new Response(response.body, response);
    if (isQuery) {
      res.headers.set('Cache-Control', 'public, max-age=60');
    }
    return res;
  },
};
# Deploy
wrangler deploy workers/twenty-proxy.ts --name twenty-proxy
wrangler secret put TWENTY_API_URL   # https://api.twenty.com
wrangler secret put TWENTY_API_TOKEN # your API key from Twenty settings

Integration with 2nth.ai

Once deployed, TwentyCRM connects to the 2nth.ai platform at multiple points:

Lead capture

Developer portal enquiry forms create leads in TwentyCRM via API. Partner, interest type, and source tracked automatically.

Token billing

Paystack webhook → token credit → TwentyCRM deal stage update. Revenue tracked per partner, per skill.

Partner pipeline

Partner onboarding stages managed in TwentyCRM. Auto-advances on first PR, first skill published, first customer.

Skill usage analytics

Monthly skill usage reports pushed to TwentyCRM as activities. Revenue share calculations attached to partner deals.
